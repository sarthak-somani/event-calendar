# Data Schema Documentation

`events.json` is the single data file shared between the backend and the frontend. It is generated by `process_events.py` and consumed by `script.js`.

---

## Structure

The file contains a JSON array. Each element is an event object compatible with the [FullCalendar event object](https://fullcalendar.io/docs/event-object) format.

```json
[
  {
    "title": "Event Title",
    "start": "YYYY-MM-DDTHH:MM:SS",
    "end": "YYYY-MM-DDTHH:MM:SS",
    "extendedProps": {
      "description": "Detailed description of the event.",
      "venue": "Location or room name",
      "organisingBody": "Department, club, or group name",
      "contact": "Email address, phone number, or website"
    }
  }
]
```

### Field reference

| Field | Type | Required | Notes |
|---|---|---|---|
| `title` | string | **Yes** | Displayed on the calendar tile and popup header. Events without a title are discarded by the backend. |
| `start` | string | **Yes** | ISO 8601 — `YYYY-MM-DDTHH:MM:SS`. Events without a start date are discarded. |
| `end` | string | No | ISO 8601. Defaults to the same as `start` if Gemini does not extract an end time. |
| `extendedProps.description` | string \| null | No | Free-text description. Displayed in the event popup. |
| `extendedProps.venue` | string \| null | No | Physical location. Displayed in the event popup. |
| `extendedProps.organisingBody` | string \| null | No | Name of the organising department or group. |
| `extendedProps.contact` | string \| null | No | Contact info (email, phone, URL, etc.). |

### Deduplication key
The backend uses `(title, start)` as a composite key when deciding whether to append a new event. An event is skipped if an identical `(title, start)` pair already exists in `events.json`.

---

## Example

```json
{
    "title": "Central Library Orientation 2025",
    "start": "2025-08-04T10:00:00",
    "end": "2025-08-08T10:00:00",
    "extendedProps": {
        "description": "Detailed Library Orientation Program for new entrants and senior students interested in learning more about library resources and services.",
        "venue": "Conference Hall, 2nd Floor, Central Library",
        "organisingBody": "Central Library",
        "contact": "+91-22-2576-8936 / 8926"
    }
}
```

---

## Notes

- **Do not edit `events.json` manually.** It is overwritten by the backend script on every run that finds new events.
- All string values come from AI extraction and may occasionally be `null` if the model could not identify the field. The frontend handles `null` gracefully by showing a "Not available" fallback.
- The file grows indefinitely as new events are appended. Old past events are not pruned automatically.
